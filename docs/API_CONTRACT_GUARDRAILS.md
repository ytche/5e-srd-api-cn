# API 契约门禁与发布检查清单

## 1. PR 门禁（必须满足）

适用于任何会影响 `docs/API_CONTRACT.md` 的后端改动。

1. 已运行 `npm run test:contract` 且通过
2. 已运行 `npm run test:unit` 且通过
3. 若涉及字段变更：
   - 已补模型字段定义
   - 已补/更新契约测试
   - 已更新 `docs/API_CONTRACT.md`（如有映射，更新映射表）
4. 若涉及错误处理变更：
   - 400/404/500 语义测试已覆盖
   - 404 返回统一 JSON：`{ "error": "Not found" }`
5. 若涉及详情结构变更：
   - 已声明兼容策略（保留字段/映射字段/破坏性变更）

---

## 2. 发布前检查（Release Checklist）

1. CI 三类任务通过：
   - `lint`
   - `unit`
   - `contract`
2. 与契约相关的变更已写入 release note（行为变化、字段变化、错误语义）
3. 与前端约定方完成一次联调确认（至少覆盖 3 个关键资源）
4. 监控/告警确认：
   - 404 比例和 500 比例告警阈值无异常

---

## 3. 破坏性变更判定

以下任一情况属于破坏性变更，必须在 PR 和发布说明中明确：

1. 删除字段或重命名字段且无映射兼容
2. 更改错误码语义（如 404 改 200）
3. 变更列表响应结构（非 `{ count, results }`）
4. 更改详情响应主对象结构

---

## 4. 建议执行顺序

1. 先改模型/控制器
2. 再改契约测试
3. 最后更新文档
4. 运行 `test:contract` 与 `test:unit`
5. 提交 PR
