# 车卡系统架构与部署设计

> 本文档描述 5e-srd-api、未来业务后端与前端（character-builder）之间的架构分工、请求设计与部署方式。维护位置：5e-srd-api 仓库。

---

## 1. 系统角色

| 系统 | 职责 | 数据 |
|------|------|------|
| **5e-srd-api**（本项目） | 只读 5e 中英数据接口 | SRD 参考数据（种族、职业、法术等），无用户、无角色存储 |
| **业务后端**（未来新建） | 用户管理、角色云端同步等 | 用户、角色等业务数据 |
| **character-builder**（小程序/H5） | 车卡流程、拉 5e 数据、存读角色（本地或云端） | 前端状态 + 可选云端 |

---

## 2. 请求设计（谁调谁）

| 请求类型 | 发起方 | 调用对象 | 说明 |
|----------|--------|----------|------|
| 种族/职业/法术等 5e 参考数据 | 前端 | **5e-srd-api** | 直连，按 `API_CONTRACT.md` 契约 |
| 用户登录/注册/会话 | 前端 | 业务后端 | 仅与业务后端通信 |
| 角色列表/详情/保存/删除（云端） | 前端 | 业务后端 | 角色数据仅存业务后端 |
| 业务后端需要「种族名/职业名」等 | **业务后端** | **5e-srd-api**（内网） | 服务端内调，不经过前端 |

原则：

- 5e-srd-api 保持单一职责，不接用户、不存角色。
- 业务后端需要 5e 数据时自行内调 5e-srd-api（如解析 index→name），不要求前端多一次请求。

## 4. 域名与配置建议

- **5e-srd-api**：独立域名或路径，例如 `https://api-5e.yourdomain.com` 或 `https://yourdomain.com/5e-api`
- **业务后端**：独立域名或路径，例如 `https://api.yourdomain.com` 或 `https://yourdomain.com/api`

前端配置两个 Base URL（或一个 Base + 一个 5e Base）：

- `SRD_API_BASE` → 5e-srd-api（只读 5e 数据）
- `APP_API_BASE` → 业务后端（用户、角色同步等）

微信小程序需在「request 合法域名」中配置实际请求的域名（见下节）。

---

## 5. 跨域与访问方式

### 5.1 微信小程序

小程序运行在微信客户端，**不受浏览器同源策略约束**，没有 CORS 问题。

- 在微信公众平台 → 小程序 → 开发管理 → 开发设置 → **服务器域名**中，将两个后端域名加入「request 合法域名」即可。
- 无需在 5e-srd-api 或业务后端配置 CORS 来满足小程序请求。

### 5.2 H5 / Web 版本

若同一套前端有 H5/Web 版本，浏览器会存在跨域，可选：

- **方案 A**：两个后端均返回 CORS 头，允许前端页面所在域名。
- **方案 B（推荐）**：用 Nginx 做反向代理，对外只暴露一个域名，按路径转发到两个后端，浏览器只请求同一 origin，无跨域。

---

## 6. Nginx 反向代理示例（H5 单域名）

对外统一域名 `https://api.yourdomain.com`，按路径映射到两个后端：

```nginx
# 5e 数据 → 5e-srd-api
location /5e/ {
    proxy_pass http://5e-srd-api:3000/api/2014/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}

# 业务接口 → 业务后端
location /api/ {
    proxy_pass http://character-service:8080/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

前端只配一个 Base URL：`https://api.yourdomain.com`。例如：

- 5e 数据：`https://api.yourdomain.com/5e/races` → 转发到 5e-srd-api
- 业务接口：`https://api.yourdomain.com/api/characters` → 转发到业务后端

小程序也可统一走该域名，只需在微信后台配置此一个合法域名。

---

## 7. 与契约文档的关系

- 5e 数据接口格式以 [API_CONTRACT.md](./API_CONTRACT.md) 为准；前端与业务后端调用 5e-srd-api 时均遵循该契约。
- 业务后端的接口与数据格式由业务后端项目单独定义，不在本仓库维护。

---

## 8. 小结

- **5e-srd-api**：仅提供 5e 中英数据，部署后前端/业务后端按需直连或内网调用。
- **业务后端**：负责用户与角色等业务，需要 5e 数据时内调 5e-srd-api。
- **小程序**：配置合法域名即可，无 CORS 问题；**H5** 可通过 Nginx 单域名代理避免跨域。
