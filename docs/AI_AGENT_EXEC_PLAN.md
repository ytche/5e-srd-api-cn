# AIAgent 执行计划（基于 API 后端开发计划）

## 1. Agent 目标

AIAgent 目标是将 `API_BACKEND_DEV_PLAN.md` 中的阶段任务自动化执行为可审阅的最小增量 PR。

---

## 2. 执行原则

- 每次仅处理一个阶段，避免大批量混改
- 每个阶段都包含：代码修改 + 单测 + 文档更新
- 每个阶段单独提交（中文 commit）
- 提交前必须运行 `npm run test:unit`

---

## 3. 工作流分解

## Step 0：准备与基线

动作：

1. 读取：`docs/API_CONTRACT.md`、`docs/API_BACKEND_DEV_PLAN.md`
2. 扫描当前实现差异（模型字段、错误处理、映射、测试覆盖）
3. 输出差异清单到 PR 描述草稿

产出：

- 差异列表（可执行任务清单）

---

## Step 1：阶段 A 自动实现（模型补齐）

动作：

1. 批量补充 2014 模型 `name_en/desc_en`
2. 补齐 `Race` 的 `starting_proficiencies` / `starting_proficiency_options`
3. 补齐 `Choice.desc_en`
4. 运行单测并修复编译/类型问题

产出：

- Commit 1（中文）
- 模型层变更说明

质量门禁：

- `npm run test:unit` 通过

---

## Step 2：阶段 B 自动实现（错误语义）

动作：

1. 增加统一 JSON 404 middleware
2. 对未命中/父资源不存在路径补齐 404 处理
3. 补充对应控制器或集成测试

产出：

- Commit 2（中文）
- 错误语义对齐说明

质量门禁：

- 400/404/500 场景测试通过

---

## Step 3：阶段 C 自动实现（字段映射）

动作：

1. 将映射规则集中（建议独立 `mapper` 工具）
2. 应用到相关详情接口
3. 增加映射单测

产出：

- Commit 3（中文）
- 字段映射表（文档）

质量门禁：

- 映射测试通过

---

## Step 4：阶段 D 自动实现（契约测试）

动作：

1. 新增 `src/tests/contract/` 契约测试集
2. 覆盖结构、字段、语义
3. 纳入 CI 命令

产出：

- Commit 4（中文）
- 契约测试说明

质量门禁：

- 契约测试可稳定复现失败/成功

---

## Step 5：阶段 E 自动实现（CI 契约守卫）

动作：

1. 增加 `test:contract` 命令
2. CI 新增独立 `contract` job
3. 验证契约测试在本地与 CI 指令一致

产出：

- Commit 5（中文）
- CI 守卫说明

质量门禁：

- `npm run test:contract` 通过

---

## Step 6：阶段 F 自动实现（PR/发布门禁）

动作：

1. 增加契约门禁文档（PR + Release checklist）
2. 更新 PR 模板，强制填写契约验证项
3. 将门禁文档纳入契约文档体系

产出：

- Commit 6（中文）
- 契约门禁执行规范

质量门禁：

- 模板和文档可直接用于团队流程

---

## 4. Agent 任务模板

每次执行采用以下模板：

1. `目标`：本次阶段与验收标准
2. `改动文件`：明确路径列表
3. `验证命令`：至少包含 `npm run test:unit`
4. `风险`：是否存在兼容性影响
5. `提交信息`：中文、可读、可追溯

---

## 5. 提交策略

- 分阶段提交，避免单次提交过大
- 提交标题格式：`阶段X：<动作>`
- 示例：
  - `阶段A：补齐2014模型双语字段声明`
  - `阶段B：统一404与500错误语义`
  - `阶段C：实现Subrace字段契约映射`
  - `阶段D：新增API契约回归测试`

---

## 6. 终态定义

当满足以下条件时，Agent 任务完成：

1. API 行为与 `API_CONTRACT.md` 一致
2. 契约测试可阻断回归
3. 单测全绿
4. 文档与实现同步
